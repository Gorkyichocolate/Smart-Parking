# pkg — Общие библиотеки проекта

## 📌 Описание

Папка `pkg` содержит переиспользуемые компоненты, которые используются в нескольких микросервисах системы Smart Parking.

Здесь находится общий инфраструктурный код, который помогает избежать дублирования и обеспечивает единый подход к разработке.

> ❗ Важно: в `pkg` не должно быть бизнес-логики.

---

## 🎯 Назначение

* Централизация общего кода
* Повторное использование между сервисами
* Снижение дублирования
* Упрощение поддержки и масштабирования

---

## 📁 Структура

```id="pkgstruct1"
pkg/
├── config/        # Загрузка конфигурации
├── logger/        # Логирование
├── middleware/    # gRPC middleware (перехватчики)
├── jwt/           # Работа с JWT
├── errors/        # Общие ошибки
├── utils/         # Вспомогательные функции
├── rabbitmq/      # Работа с RabbitMQ
├── postgres/      # Подключение к PostgreSQL
├── redis/         # Подключение к Redis
└── metrics/       # Метрики (Prometheus)
```

---

## 🧩 Описание модулей

### 🔧 config/

Отвечает за загрузку конфигурации приложения (например, из `.env` или переменных окружения).

---

### 🪵 logger/

Предоставляет единый механизм логирования для всех сервисов (например, через zap или logrus).

---

### 🧱 middleware/

Содержит gRPC-перехватчики:

* аутентификация
* логирование
* сбор метрик

---

### 🔐 jwt/

Отвечает за:

* генерацию JWT токенов
* валидацию токенов
* извлечение данных пользователя

---

### ❗ errors/

Содержит общие ошибки, используемые в разных сервисах.

---

### 🧰 utils/

Вспомогательные функции:

* хеширование
* генерация ID
* работа со строками

---

### 🐇 rabbitmq/

Инкапсулирует работу с RabbitMQ:

* создание подключения
* отправка сообщений (publisher)
* получение сообщений (consumer)

---

### 🗄 postgres/

Создаёт и управляет подключением к PostgreSQL.

---

### ⚡ redis/

Инициализирует Redis-клиент.

---

### 📊 metrics/

Содержит метрики Prometheus:

* количество запросов
* задержки
* ошибки

---

## 💻 Пример использования

```go id="pkgexample1"
cfg := config.Load()

db := postgres.New(cfg.DBUrl)
rdb := redis.New(cfg.RedisUrl)

logger := logger.New()
```

---

## 📏 Правила использования

* ❌ Не добавлять бизнес-логику
* ❌ Не добавлять код, завязанный на конкретный сервис
* ✅ Писать только универсальные и переиспользуемые компоненты
* ✅ Стараться избегать дублирования кода

---

## 🧠 Когда использовать pkg/

Используй `pkg`, если:

* код нужен в нескольких сервисах
* это инфраструктурный код
* он не зависит от конкретной предметной области

---

## 🚫 Когда НЕ использовать pkg/

Не используй `pkg` для:

* бизнес-логики (`internal/usecase`)
* моделей (`internal/domain`)
* специфичной логики сервиса
